Distribution is the whole game for a bottom-up dev tool. Rambo meets developers where they already are.

## 1. The standalone desktop app
The primary surface — fast, native, offline-first. Direct download + auto-update.

## 2. The CLI (npm / brew)
The [deterministic CLI](/rambo/features/09-agent-pillar-mcp-cli/) is both a CI/CD runner and a distribution surface. `npm i -g` / `brew install` puts Rambo in front of developers who live in the terminal.

## 3. MCP registries & agent ecosystems
The first-party **MCP server** means AI coding agents (Claude Code, Cursor, Copilot) can *recommend and drive* Rambo. As agent-driven development grows, being the default API client in agent workflows is a compounding channel no incumbent owns cleanly.

## 4. The VS Code Marketplace
Thunder Client proved this channel at scale (**7.1M+ installs**). A VS Code extension surface is a high-volume, low-friction on-ramp — sequenced after the standalone app lands (P2).

## 5. Open, shareable artifacts
Git-native collections mean teams share Rambo setups by simply pushing a repo — every shared repo is an organic advertisement, with zero infra cost to us.

## 6. The trust narrative as content
"Your data is yours," the secret-leakage story, and the anti-bloat angle are content marketing that writes itself — anchored to [validated pain](/rambo/opportunity/customer-pain/) developers already feel.

:::tip
Each channel is low- or zero-CAC and reinforces the others: the CLI feeds CI, MCP feeds agents, git feeds team spread, and the trust story feeds them all.
:::
