## Motion

The best launch motion is bottom-up:

1. Import from Postman/Insomnia/Bruno/OpenAPI/curl.
2. Show a no-account local workspace immediately.
3. Demonstrate one painful workflow solved better: LLM stream debugging or secret-safe collaboration.
4. Demonstrate request diagnostics: DNS/TCP/TLS/timing/proxy evidence next to the failed request.
5. Publish CLI/MCP examples for Codex, Claude Code, Cursor, VS Code, and CI.
6. Convert teams through sync, review, and encrypted shared secrets.
7. Convert enterprise through storage control, SSO/SCIM/RBAC/audit, and self-hosting.

## Counter-positioning

| Competitor wound | Rambo message |
|---|---|
| Postman pricing/free-team erosion | "Small teams can collaborate without immediately paying the Postman tax." |
| Postman cloud/trust | "Your workspace is local until you choose otherwise." |
| Insomnia account backlash | "No account required for local work, by design." |
| Bruno rough edges | "Git-native trust with commercial polish and enterprise support." |
| Yaak overlap | "Offline-first plus deeper LLM runtime debugging and team governance." |
| Network failure ambiguity | "Rambo shows whether the failure is DNS, TCP, TLS, proxy, server, or request logic." |

## Channels

- GitHub releases and comparison docs.
- VS Code/Cursor integration later, not as MVP dependency.
- MCP registry and agent-tooling communities.
- LLM API debugging demos with Ollama/OpenAI-compatible streams.
- Migration guides for Postman, Insomnia, Bruno, Hoppscotch.
- Security/trust content: secrets, local-first, no surprise sync.
- Troubleshooting content: "debug API failures without leaving the client."

## Switching requirement

Switching is only credible if import shows:

- imported object counts;
- unsupported fields;
- script translation warnings;
- variable/secret safety warnings;
- before/after diff;
- rollback/export path.
