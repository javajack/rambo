## The sizing trap

Do not pitch Rambo against the full API-management TAM. API management includes gateways, API security, developer portals, monetization, analytics, governance, and lifecycle platforms. Rambo is initially an API client/testing/workflow product, so the defensible market is a narrower tools slice.

## Practical SAM

The practical initial SAM is developers and teams that already use Postman/Insomnia/Bruno/Yaak/Hoppscotch/Thunder Client for:

- manual API exploration;
- saved collections/environments;
- local and CI collection runs;
- protocol debugging;
- team sharing;
- API docs/spec import;
- LLM/API streaming diagnosis.

The paid SAM is smaller: teams willing to pay for collaboration, sync, governance, and enterprise packaging. Individual devs are acquisition, not necessarily revenue.

## Implication

The company should be planned as a capital-efficient developer-tool business first. The evidence supports a plausible $1M-$10M ARR path if execution is strong, but not a guaranteed venture-scale outcome.
