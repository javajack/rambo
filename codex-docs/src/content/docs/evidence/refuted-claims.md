## Refuted or weakened

| Claim | Status | Why |
|---|---|---|
| "MCP is a unique Rambo differentiator" | Weakened | Postman has a first-party MCP Server with hosted/local options and 100+ tools; Yaak has an MCP plugin |
| "Insomnia is simply cloud-default now" | Weakened | Current Insomnia pricing advertises local/Git/cloud storage, E2EE, free Git Sync up to 3 users, and Enterprise storage mandates |
| "Yaak is a minor competitor" | Refuted | Yaak is offline-first, MIT, Tauri/Rust/React, multi-protocol, has encrypted secrets, OS keychain, and MCP |
| "Low price alone can win" | Refuted | Bruno, Hoppscotch, Thunder Client, and Yaak are already cheap/free; switching depends on import, secrets, trust, and team workflows |
| "AI API client is enough as a category" | Weakened | Postman and Insomnia market AI-native features; Rambo needs a specific LLM runtime/debugging surface |
| "Self-hosting removes compliance burden" | Weakened | It reduces data-processing exposure but buyers still require security, contracts, support, and vendor-risk diligence |
| "Free-up-to-5 is proven optimal" | Weakened | Evidence supports small-team cap direction, not the exact number |

## Preserved claims

| Claim | Status | Why |
|---|---|---|
| Forced-account/cloud-lock-in pain is real | Preserved | Multiple primary public issues and recurring community sentiment |
| Secrets/local storage are trust-critical | Preserved | Public leaks plus Postman docs now emphasize Vault/local secrets |
| LLM streaming/debugging is underserved | Preserved/narrowed | Specific evidence from Postman #13537 and Bruno #7835 |
| Local-first/no-account is strategically coherent | Preserved | It directly counters incumbent trust failures and supports low infra |
